# YOUR TASK FOR THIS CALL — Per-Build Scoring

You are now scoring ONE concept against the calibrated weights in the persona above. This is NOT a pairwise comparison. You will see:

1. **Brief context** — the product / sample data / outcome the build was supposed to deliver
2. **This concept's approach doc** — what the designer SAID they'd build
3. **Other concepts' approach summaries** — for the Distinctiveness dimension only
4. **Walker journey screenshots for THIS concept** — initial state + state-change frames captured by clicking through the interactive prototype
5. **Moodboard references** — the visual direction the build was supposed to match

## Required protocol

Follow the persona's Scoring Process. Specifically:

1. Read the approach doc first. Note the techniques the designer promised.
2. Look at the journey screenshots cold. Write a 1-sentence gut reaction.
3. Run the Anti-Pattern checklist. Flag every instance you can see.
4. Verify approach-doc compliance technique by technique — implemented? visible on screen?
5. Compare against siblings on Distinctiveness (palette family, composition pattern, primary visual technique). Two siblings with the same palette + same technique = each one's Distinctiveness drops hard.
6. Check the Technical Execution binary gate:
   - If the build does not render (blank journey, fatal errors, missing fonts) → clamp `weighted_total` to ≤3.0 regardless of category scores.
7. Score each weighted dimension starting from 5.0. Earn points up. Don't start high and dock.
8. If you flag AI Slop, clamp `weighted_total` to ≤4.0.
9. Produce the Priority Fixes list. These will be fed back to the builder for the next polish iteration, so they must be CONCRETE and ACTIONABLE — "tighten the type" is useless; "drop body to 14px, drop tracking to -0.01em, add a 600-weight subhead above each item" is useful.
