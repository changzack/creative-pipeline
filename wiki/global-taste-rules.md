# Global Taste Rules

Universal taste rules. These apply to EVERY pipeline run regardless of artifact
type — cards, games, onboarding flows, posters, dashboards, anything.
Brief-specific lessons live in the brief's sibling `LEARNINGS.md`, not here.

Keep this file under ~1.5K chars. If a rule only applies to one brief, it does
not belong here.

## The 6 rules

1. **No AI slop.** The output must feel hand-crafted, not template. Clean +
   polished + soulless is the #1 failure mode. A rough but creative concept
   beats a polished but generic one every time.

2. **Real craft beats polish.** Rough edges, hand-tuned details, and visible
   intentionality read as "designer made this." Default-tailwind smoothness
   reads as "AI made this." Lean toward the former.

3. **Visual hierarchy is non-negotiable.** One thing must own the eye on first
   glance. Equal-weight grids of equal-weight elements are dead on arrival.

4. **Texture and depth separate artifact from template.** Flat color blocks +
   sans-serif + 16px radius = template. Grain, layered shadows, edge work,
   typographic contrast, real materials = artifact.

5. **Push for novel techniques over safe defaults.** WebGL, SVG filters, CSS
   3D, generative patterns, scroll-driven motion, real physics > rounded
   cards with hover states. If you can describe your concept in one
   ten-word sentence with no proper nouns, it's not ambitious enough.

6. **The metaphor must be visible, not just described.** If the brief asks
   for "stock-market vibe," the chart must BE part of the design language —
   not a stock-market label slapped on a dashboard. Concept and form fuse.
